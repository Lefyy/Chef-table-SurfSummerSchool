# Feature: Авторизация клиента по телефону и SMS-коду

## Описание

Приложение поддерживает вход клиента по номеру телефона и одноразовому 4-значному SMS-коду. В local/dev окружении используется mock-код `1234`, что позволяет проверять сценарий без реального SMS-шлюза.

## Основной сценарий

1. Клиент открывает `/login`.
2. Вводит телефон в международном формате.
3. Получает экран ввода кода.
4. Вводит корректный код.
5. Приложение устанавливает защищённую cookie-сессию и перенаправляет клиента на расписание или исходный защищённый URL.

## Правила и ограничения

- Телефон должен содержать 10–15 цифр и может начинаться с `+`.
- SMS-код состоит ровно из 4 цифр.
- Небезопасные внешние redirect-значения не используются.
- Logout удаляет локальную cookie-сессию и возвращает пользователя на экран входа.
