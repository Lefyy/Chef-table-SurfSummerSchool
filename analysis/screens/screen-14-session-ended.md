# Сессия завершена

## Назначение
Сообщить клиенту, что он вышел из системы, и предложить повторный вход.

## Пользовательские роли
- Клиент.

## Источники требований
- FR-25; US-21; UC-09; NFR-06.

## Когда отображается
- После выхода из системы.
- При ошибке отзыва токена, если локальная сессия всё равно удалена.

## Откуда можно перейти
- Bottom Sheet «Подтверждение выхода».
- «Профиль» после успешного выхода. **Inferred**.

## Куда можно перейти
- «Вход: телефон».

## Основной пользовательский сценарий
1. Система удаляет локальную сессию/токен.
2. Показывает нейтральное сообщение о выходе.
3. Клиент может войти снова.

## Состав экрана
- Сообщение: «Вы вышли из системы. Повторите вход при следующем использовании» для случая ошибки отзыва токена.
- CTA «Войти».

## Действия пользователя
- Перейти к входу.

## Связанные Bottom Sheets
- `bottom-sheet-05-logout-confirmation.md`.

## Состояния
- Default/Success: сессия завершена.
- Loading: не применимо.
- Empty: не применимо.
- Error: если локальную сессию удалить не удалось, показать «Ошибка сервера» и оставить клиента в текущем состоянии; этот экран не должен открываться.
- Disabled: не применимо.
- Offline: не описано. **Missing**.

## Валидация
- Нет пользовательского ввода.

## Ограничения
- После выхода защищённые сценарии недоступны.

## Edge Cases
- Бэкенд не отозвал токен, но локальная сессия удалена.

## Inferred
- Экран может быть заменён переходом на экран входа с toast-сообщением, но отдельное состояние полезно для UX-дизайна.

## Missing
- Должен ли после выхода быть отдельный экран или автоматический редирект на вход.
