# Бронирование: прокат экипировки

## Назначение
Позволить клиенту выбрать доступный прокат экипировки для брони.

## Пользовательские роли
- Клиент.

## Источники требований
- Domain: 5.3, RentalItem; BR-09; FR-12; US-09; UC-04; NFR-08.

## Когда отображается
- После нажатия «Забронировать» в деталях класса.

## Откуда можно перейти
- «Детали класса».

## Куда можно перейти
- «Бронирование: аллергии».
- Назад к «Детали класса» или предыдущему шагу. **Inferred**.

## Основной пользовательский сценарий
1. Система показывает список позиций проката.
2. Клиент выбирает нужные позиции или пропускает выбор.
3. Клиент переходит к аллергиям.

## Состав экрана
- Контекст выбранного класса. **Inferred**.
- Список RentalItem: название, цена, доступность.
- Недоступные позиции при `stockAvailable = 0`.
- CTA «Далее».
- Возможность пропустить без выбора. **Inferred**.

## Действия пользователя
- Выбрать/снять позиции проката.
- Перейти дальше.
- Вернуться назад.

## Связанные Bottom Sheets
- Нет.

## Состояния
- Default: список проката.
- Loading: загрузка/проверка доступности.
- Empty: прокат отсутствует или недоступен. **Missing** для текста.
- Error: «Ошибка сервера».
- Success: выбор сохранён в текущей воронке.
- Disabled: позиции с `stockAvailable = 0`; CTA может быть доступен без выбора. **Inferred**.
- Offline: не описано. **Missing**.

## Валидация
- Нельзя выбрать позицию с `stockAvailable = 0`.
- Цена определяется бэкендом и не редактируется.

## Ограничения
- Прокат списывается из `stockAvailable` при бронировании бэкендом.
- В одной брони бронируется 1 место.

## Edge Cases
- Выбранная позиция стала недоступна до подтверждения.
- Нет доступных позиций проката.

## Inferred
- Выбор проката необязателен, так как требования говорят «выбор проката», но не делают его обязательным.

## Missing
- Можно ли выбрать количество одной позиции, список всех позиций кроме ножей/фартука, текст пустого состояния.
