# История бронирований

## Назначение
Показать клиенту историю бронирований, статусы классов и оплаты.

## Пользовательские роли
- Клиент.

## Источники требований
- Domain: 5.7; FR-23, FR-29; US-19, US-24; UC-07; NFR-04, NFR-07, NFR-19.

## Когда отображается
- Из профиля при выборе «История».

## Откуда можно перейти
- «Профиль».

## Куда можно перейти
- «Детали бронирования».
- «Оценка шефа» для подходящих завершённых посещённых броней. **Inferred via details or direct CTA**.

## Основной пользовательский сценарий
1. Клиент открывает историю.
2. Система получает список бронирований.
3. Клиент просматривает статусы и открывает нужную запись.

## Состав экрана
- Список бронирований.
- Для каждой записи: программа, дата/время, шеф/адрес **Inferred**, статус брони, статус оплаты «Не оплачено»/«Оплачено».
- CTA/индикатор «Оценить шефа» для подходящих броней. **Inferred**.

## Действия пользователя
- Открыть бронь.
- Перейти к оценке, если доступно.
- Вернуться в профиль.

## Связанные Bottom Sheets
- Нет.

## Состояния
- Default: список истории.
- Loading: загрузка истории.
- Empty: бронирований нет. **Missing** для точного текста.
- Error: «Ошибка сервера».
- Success: история загружена.
- Disabled: действия недоступны во время загрузки. **Inferred**.
- Offline: не описано. **Missing**.

## Валидация
- Доступ только авторизованному клиенту.
- Статус оплаты только из бэкенда, только «Не оплачено» или «Оплачено».

## Ограничения
- Онлайн-оплата не предоставляется.

## Edge Cases
- Исторические данные отсутствуют для старых периодов.
- Бронь отменена студией — показать статус.

## Inferred
- Нужна отдельная карточка/строка брони для перехода к деталям.

## Missing
- Пагинация/сортировка истории, период хранения, точный empty-текст.
