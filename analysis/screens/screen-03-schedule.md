# Расписание классов

## Назначение
Показать доступные классы и дать клиенту выбрать слот для просмотра деталей.

## Пользовательские роли
- Клиент.

## Источники требований
- Domain: 5.2; BR-07, BR-08; FR-05, FR-06, FR-07, FR-08, FR-09; US-03–US-06; UC-02; NFR-02, NFR-14.

## Когда отображается
- После успешной авторизации.
- При переходе из навигации к расписанию. **Inferred**.

## Откуда можно перейти
- «Вход: SMS-код».
- «Детали класса», «Бронь подтверждена», «Профиль». **Inferred**.

## Куда можно перейти
- «Детали класса» по карточке слота.
- Bottom Sheet «Фильтры расписания».
- «Профиль». **Inferred**.

## Основной пользовательский сценарий
1. Система загружает расписание на ближайшие 7 дней.
2. Клиент просматривает список карточек.
3. При необходимости открывает фильтры и уточняет выдачу.
4. Клиент выбирает интересующий слот.

## Состав экрана
- Заголовок «Расписание».
- Список/лента дней и карточек слотов.
- CTA/иконка фильтров.
- Индикатор активных фильтров. **Inferred**.
- Карточка слота: время, программа, шеф, свободные места, иконка духовки, иконка проката.
- Пустое состояние: «Пока нет доступных классов.»

## Действия пользователя
- Просмотреть расписание.
- Открыть фильтры.
- Сбросить/применить фильтры через Bottom Sheet.
- Открыть слот.

## Связанные Bottom Sheets
- `bottom-sheet-01-schedule-filters.md`.

## Состояния
- Default: карточки ближайших 7 дней.
- Loading: загрузка расписания/применение фильтров.
- Empty: «Пока нет доступных классов.»
- Error: «Ошибка сервера».
- Success: список обновлён.
- Disabled: карточки недоступных/отменённых слотов не должны вести к бронированию. **Inferred**.
- Offline: не описано. **Missing**.

## Валидация
- Фильтры применяются по И-логике.
- Диапазон просмотра за пределами 7 дней ограничен датами до конца следующей недели включительно.

## Ограничения
- Авторизация обязательна.
- Данные и цены/доступность приходят из бэкенда.

## Edge Cases
- Все результаты отфильтрованы.
- Свободные места изменились после загрузки списка.
- Шеф без среднего рейтинга: показывать без рейтинга.

## Inferred
- Нужен явный индикатор активных фильтров и возможность сброса.

## Missing
- Сортировка, группировка по дням, формат карточки и навигационная модель.
