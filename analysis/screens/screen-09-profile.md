# Профиль

## Назначение
Дать клиенту доступ к истории, сохранённым аллергиям и выходу из системы.

## Пользовательские роли
- Клиент.

## Источники требований
- Domain: 5.7; FR-23, FR-24, FR-25; US-19, US-20, US-21; UC-07, UC-08, UC-09; NFR-04, NFR-06, NFR-07.

## Когда отображается
- При выборе раздела профиля авторизованным клиентом.

## Откуда можно перейти
- «Расписание классов». **Inferred**.
- Другие защищённые экраны через навигацию. **Inferred**.

## Куда можно перейти
- «История бронирований».
- «Мои аллергии».
- Bottom Sheet «Подтверждение выхода».

## Основной пользовательский сценарий
1. Клиент открывает профиль.
2. Выбирает историю, мои аллергии или выход.

## Состав экрана
- Телефон клиента. **Inferred from profile.phone**.
- Навигационные элементы: «История», «Мои аллергии».
- Кнопка «Выйти».

## Действия пользователя
- Открыть историю.
- Открыть сохранённые аллергии.
- Инициировать выход.

## Связанные Bottom Sheets
- `bottom-sheet-05-logout-confirmation.md`.

## Состояния
- Default: профиль доступен.
- Loading: загрузка данных профиля.
- Empty: профиль без истории/аллергий отражается в дочерних экранах.
- Error: «Ошибка сервера».
- Success: данные профиля загружены.
- Disabled: действия во время загрузки/выхода. **Inferred**.
- Offline: не описано. **Missing**.

## Валидация
- Доступ только после авторизации.

## Ограничения
- Редактирование профиля не описано.

## Edge Cases
- Сессия истекла: перевод к авторизации. **Inferred**.

## Inferred
- Профиль показывает телефон, потому что сущность ClientProfile содержит `phone`.

## Missing
- Дизайн навигации, возможность редактировать телефон/профиль, срок жизни сессии.
