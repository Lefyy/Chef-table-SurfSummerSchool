# Мои аллергии

## Назначение
Показать сохранённые аллергии клиента только для просмотра.

## Пользовательские роли
- Клиент.

## Источники требований
- Domain: 5.7; FR-14, FR-24; US-20; UC-08; NFR-04, NFR-07, NFR-11.

## Когда отображается
- Из профиля при выборе «Мои аллергии».

## Откуда можно перейти
- «Профиль».

## Куда можно перейти
- Назад в профиль.

## Основной пользовательский сценарий
1. Клиент открывает раздел.
2. Система получает сохранённые аллергии.
3. Клиент просматривает список без редактирования.

## Состав экрана
- Заголовок «Мои аллергии».
- Список сохранённых аллергенов.
- Empty-состояние при отсутствии аллергий.
- Пояснение, что изменение возможно через сценарий бронирования. **Inferred from UC exception**.

## Действия пользователя
- Просмотреть список.
- Вернуться.

## Связанные Bottom Sheets
- Нет.

## Состояния
- Default: список сохранённых аллергий.
- Loading: загрузка.
- Empty: сохранённых аллергий нет.
- Error: «Ошибка сервера».
- Success: данные загружены.
- Disabled: редактирование отсутствует.
- Offline: не описано. **Missing**.

## Валидация
- Нет пользовательского ввода.

## Ограничения
- Редактирование аллергий в профиле не предусмотрено.

## Edge Cases
- Аллергии были сохранены в предыдущей брони, но справочник изменился. **Missing**.

## Inferred
- Нужен текст, объясняющий отсутствие редактирования.

## Missing
- Точный empty-текст, правила удаления/обновления сохранённых аллергий.
