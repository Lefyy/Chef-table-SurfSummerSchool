# Системная ошибка сервера

## Назначение
Показать универсальную ошибку для операций бэкенда без специального описания.

## Пользовательские роли
- Клиент.

## Источники требований
- FR-30; NFR-10; исключения UC-01–UC-11.

## Когда отображается
- При ошибке бэкенда без более конкретного сообщения на критичном экране или операции.

## Откуда можно перейти
- Любой экран, выполняющий запрос к бэкенду. **Inferred**.

## Куда можно перейти
- Повторить действие. **Inferred**.
- Вернуться назад/к расписанию. **Inferred**.

## Основной пользовательский сценарий
1. Запрос к бэкенду завершается неописанной ошибкой.
2. Система показывает «Ошибка сервера».
3. Клиент может повторить действие или вернуться.

## Состав экрана
- Текст «Ошибка сервера».
- CTA «Повторить» / «Назад». **Inferred**.

## Действия пользователя
- Повторить загрузку/операцию.
- Вернуться назад.

## Связанные Bottom Sheets
- Нет.

## Состояния
- Default/Error: сообщение ошибки.
- Loading: повторная попытка.
- Empty: не применимо.
- Success: возврат на исходный экран после успешного повтора.
- Disabled: CTA во время повтора.
- Offline: отдельный offline-сценарий не описан. **Missing**.

## Валидация
- Нет пользовательского ввода.

## Ограничения
- Для специальных ошибок использовать специальные тексты: поздняя отмена, отсутствие мест и др.

## Edge Cases
- Ошибка на фоне частично загруженного экрана может быть inline, а не отдельным экраном. **Inferred**.

## Inferred
- Нужен общий паттерн системной ошибки, чтобы не проектировать разные сообщения для каждого запроса.

## Missing
- Правила retry, timeout, offline и логирования для пользователя.
