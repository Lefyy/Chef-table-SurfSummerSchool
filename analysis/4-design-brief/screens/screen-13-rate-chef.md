# Оценка шефа

## Назначение
Позволить клиенту один раз оценить шефа после посещённого завершённого класса.

## Пользовательские роли
- Клиент.

## Источники требований
- Domain: 5.6; BR-15, BR-16; FR-26, FR-27, FR-28; US-22, US-23; UC-10; NFR-12.

## Когда отображается
- Из истории/деталей бронирования, если класс завершён, клиент посетил занятие, оценка ещё не отправлялась.

## Откуда можно перейти
- «История бронирований».
- «Детали бронирования».

## Куда можно перейти
- Назад к деталям/истории после отправки.

## Основной пользовательский сценарий
1. Клиент выбирает оценку от 1 до 5.
2. При желании вводит комментарий.
3. Отправляет оценку.
4. Система сохраняет оценку через бэкенд и больше не показывает повторную оценку для этой брони.

## Состав экрана
- Информация о классе и шефе.
- Контрол оценки 1–5 звёзд.
- Необязательное поле комментария.
- CTA «Отправить оценку».
- Подсказка, что публично отображается только средний рейтинг. **Inferred**.

## Действия пользователя
- Выбрать 1–5 звёзд.
- Ввести/не вводить комментарий.
- Отправить.
- Вернуться без отправки. **Inferred**.

## Связанные Bottom Sheets
- Нет.

## Состояния
- Default: форма оценки.
- Loading: отправка.
- Empty: оценка не выбрана.
- Error: «Ошибка сервера»; значение вне диапазона не отправлять.
- Success: оценка отправлена.
- Disabled: CTA до выбора оценки; повторная отправка после успеха.
- Offline: не описано. **Missing**.

## Валидация
- Оценка обязательна, диапазон 1–5.
- Комментарий необязателен.
- Повторная оценка/изменение оценки запрещены.

## Ограничения
- Индивидуальные комментарии не отображаются публично; доступны администраторам во внешней системе.

## Edge Cases
- Бэкенд вернул «не посетил» — экран/CTA недоступен.
- Класс ещё не завершён — экран/CTA недоступен.
- Оценка уже отправлена — экран/CTA недоступен.

## Inferred
- CTA должен быть disabled до выбора звёзд.

## Missing
- Максимальная длина комментария, точный success-текст, возможность закрыть форму с несохранённым комментарием.
