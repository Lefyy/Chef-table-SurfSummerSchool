# Вход: телефон

## Назначение
Начать авторизацию клиента по номеру телефона и запросить одноразовый SMS-код.

## Пользовательские роли
- Клиент.

## Источники требований
- Domain: 5.1, 7; BR-10; FR-01; US-01; UC-01; NFR-03, NFR-05.

## Когда отображается
- При первом открытии приложения без активной сессии.
- После выхода из системы.
- При попытке открыть защищённый сценарий без авторизации.

## Откуда можно перейти
- Прямое открытие приложения.
- Экран «Сессия завершена».
- Любой защищённый маршрут при отсутствии сессии. **Inferred**.

## Куда можно перейти
- «Вход: SMS-код» после успешной отправки кода.

## Основной пользовательский сценарий
1. Клиент вводит номер телефона.
2. Нажимает «Получить код».
3. Система отправляет запрос на отправку SMS-кода.
4. При успехе открывается экран ввода SMS-кода.

## Состав экрана
- Заголовок входа.
- Поле номера телефона.
- CTA «Получить код».
- Область ошибок/подсказок.

## Действия пользователя
- Ввести/изменить номер телефона.
- Отправить номер для получения SMS-кода.

## Связанные Bottom Sheets
- Нет.

## Состояния
- Default: пустое или предзаполненное поле телефона. **Inferred**.
- Loading: отправка запроса SMS.
- Empty: поле телефона пустое.
- Error: «Ошибка сервера» для неописанных ошибок; ошибки формата телефона — **Missing**.
- Success: код отправлен, переход к экрану SMS-кода.
- Disabled: CTA заблокирован при пустом/невалидном телефоне. **Inferred**.
- Offline: отдельное поведение не описано. **Missing**.

## Валидация
- Номер телефона обязателен. **Inferred**.
- Формат номера не указан. **Missing**.

## Ограничения
- Только русский язык.
- Авторизация только по телефону и SMS-коду; email, соцсети, биометрия вне MVP.

## Edge Cases
- Бэкенд недоступен: показать «Ошибка сервера».
- Пользователь уже авторизован и открыл экран входа: поведение не указано. **Missing**.

## Inferred
- Требуется маска телефона и блокировка CTA до ввода номера.

## Missing
- Формат телефона, тексты privacy/consent, правила обработки уже зарегистрированного/нового клиента.
