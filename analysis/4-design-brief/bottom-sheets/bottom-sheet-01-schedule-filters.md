# Фильтры расписания

## Назначение
Позволить клиенту уточнить список классов по параметрам расписания.

## Когда открывается
- По нажатию кнопки/иконки фильтров на экране расписания.

## Откуда вызывается
- `screen-03-schedule.md`.

## Содержимое
- Дата или диапазон дат.
- Программа/меню.
- Шеф.
- Уровень сложности.
- Наличие свободных мест.
- Доступность проката экипировки.
- CTA «Применить».
- CTA «Сбросить». **Inferred**.

## Действия пользователя
- Выбрать значения фильтров.
- Применить фильтры.
- Сбросить фильтры.
- Закрыть без применения. **Inferred**.

## Состояния
- Default: текущие значения фильтров.
- Loading: загрузка справочников/применение.
- Empty: нет значений в конкретном справочнике. **Missing**.
- Error: «Ошибка сервера».
- Disabled: применение во время загрузки или при недопустимом диапазоне дат.

## Валидация
- Фильтры работают по И-логике.
- За рамками ближайших 7 дней через фильтр дат можно смотреть только даты до конца следующей недели включительно.

## Edge Cases
- Комбинация фильтров даёт пустой результат.
- Справочник шефов/программ недоступен.

## Связанные экраны
- `screen-03-schedule.md`.

## Inferred
- Bottom Sheet выбран как мобильный UX-паттерн для адаптивного веба; требования прямо не называют Bottom Sheet.
- Нужен сброс фильтров и индикатор активных фильтров на экране расписания.

## Missing
- Тип контролов для каждого фильтра, сохранение фильтров, поиск внутри справочников.
