# Описание домена: Клиентское веб-приложение кулинарной студии «Шеф-стол»

## 1. Краткое резюме домена

Веб-приложение (адаптивный веб) для самостоятельной записи клиентов на кулинарные классы студии «Шеф-стол».

Приложение потребляет готовый API существующего бэкенда, не создаёт и не редактирует расписание, шефов и слоты.

Фокус системы:

- воронка бронирования;
- выбор проката экипировки;
- указание аллергий;
- оценка шефа после класса;
- история бронирований в профиле.

---

## 2. Цель системы / продукта

Снять операционную нагрузку с владельца и администратора:

- исключить ручную запись в WhatsApp и таблицы;
- предотвратить двойные брони;
- избежать пропусков из-за забывчивости;
- учитывать аллергии клиентов.

Предоставить клиентам удобный канал самостоятельной записи с телефона.

---

## 3. Участники и роли

| Роль | Описание | Взаимодействие с приложением |
|---|---|---|
| Клиент | Посетитель студии, записывающийся на классы | Основная роль: поиск слотов, бронирование, выбор проката, аллергии, отмена (>12 ч), оценка шефа, просмотр истории |
| Бэкенд (black-box) | Существующая инфраструктура: расписание, слоты, шефы, фонд проката, бизнес-логика атомарной брони | Источник истины: отдаёт каталог слотов/программ/шефов/тарифов проката; принимает брони, отмены, оценки; гарантирует отсутствие двойных броней |
| Владелец/Администратор | Управляет студией через отдельную админку (вне скоупа) | Не пользуется клиентским приложением; отмена класса выполняется в админке; приложение получает статус «Отменён студией» и пуш (пуш вне MVP) |
| Шеф | Проводит классы | Не пользуется клиентским приложением; видит оценки в админке |

> **Примечание:** в текущей поставке реализована только роль **Клиент**. Интерфейсы шефа и администратора уже существуют в инфраструктуре.

## 4. Основные сущности домена

| Сущность | Ключевые атрибуты (из API) | Примечания |
|---|---|---|
| Класс (Slot / Расписание) | `id`, `startsAt`, `endsAt` (~3 ч), `programId`, `chefId`, `maxSeats` (8 или 12), `availableSeats`, `address`, `level`, `needsOven` | Бэкенд отдаёт слоты на 7 дней вперёд |
| Программа / Меню | `id`, `title`, `description`, `level`, `menuItems[]`, `needsOven` | Фильтрация по названию и уровню |
| Шеф | `id`, `name`, `avgRating`, `specialization[]` | Фильтрация по имени |
| Бронь (Booking) | `id`, `slotId`, `clientId`, `status`, `createdAt`, `equipmentRental[]`, `allergies[]`, `paymentStatus` | Атомарное создание на стороне бэкенда |
| Прокат экипировки (RentalItem) | `id`, `name`, `price`, `stockTotal`, `stockAvailable` | Если `stockAvailable = 0`, позиция недоступна. Список - набор ножей, фартук. Цена определяется бекендом. |
| Аллергия (Allergen) | `id`, `name` | Множественный выбор, сохраняется в профиле. Список: глютен, молочные продукты, яйца, рыба, ракообразные, моллюски, арахис, орехи, соя, кунжут, сельдерей, горчица, люпин, сульфиты. |
| Оценка шефа (Rating) | `bookingId`, `chefId`, `stars`, `comment`, `createdAt` | Доступна после завершения класса |
| Профиль клиента (ClientProfile) | `phone`, `savedAllergies[]`, `bookingHistory[]` | Авторизация по телефону и SMS-коду |

## 5. Основные процессы и сценарии

### 5.1. Вход / Авторизация

1. Клиент открывает приложение и вводит телефон.
2. Бэкенд отправляет одноразовый SMS-код (4 цифры).
3. Код имеет ограниченное время жизни (5 минут) и лимит попыток ввода (3).
4. Клиент вводит код и получает JWT/сессию.

---

### 5.2. Просмотр и фильтрация расписания

Экран «Расписание» по умолчанию показывает ближайшие 7 дней.

#### Фильтры

- дата или диапазон дат;
- программа/меню;
- шеф;
- уровень сложности;
- наличие свободных мест;
- доступность проката экипировки.

#### Пустое состояние

> Пока нет доступных классов.

#### Карточка слота

Содержит:

- время;
- программу;
- информацию о шефе;
- количество свободных мест;
- иконку духовки;
- иконку проката.

---

### 5.3. Воронка бронирования

1. Клиент открывает слот.
2. Просматривает описание меню, шефа и адрес.
3. Переходит к бронированию.
4. Выбирает прокат экипировки.
5. Указывает аллергии.
6. Просматривает итоговую сводку.
7. Подтверждает бронь.
8. Бэкенд атомарно создаёт бронь со статусом **Активна**.

---

### 5.4. Отмена брони клиентом

Если до начала занятия больше 12 часов:

- отмена разрешена;
- место возвращается;
- статус меняется на «Отменена клиентом».

Если осталось 12 часов или меньше:

> Отмена недоступна менее чем за 12 часов до класса.

---

### 5.5. Отмена класса студией

Отмена выполняется через административную систему.

В приложении:

- бронь получает статус «Отменена студией»;
- отображается причина отмены;
- повторная запись на слот запрещена.

Пуш-уведомления не входят в MVP.

---

### 5.6. Оценка шефа

После завершения класса:

1. В профиле появляется кнопка «Оценить шефа».
2. Клиент ставит оценку от 1 до 5.
3. Комментарий остаётся необязательным.
4. Бэкенд пересчитывает средний рейтинг.

Публично отображается только средняя оценка.

---

### 5.7. Профиль клиента

Содержит:

- вкладку «История»;
- раздел «Мои аллергии»;
- кнопку выхода из системы.

## 6. Бизнес-правила

| ID | Правило |
|---|---|
| BR-01 | Бронь создаётся только при `availableSeats > 0`. |
| BR-02 | Максимум мест: 12 либо 8 при `needsOven = true`. |
| BR-03 | Прокат списывается из `stockAvailable`. |
| BR-04 | Отмена доступна только более чем за 12 часов. |
| BR-05 | Оплата производится только на месте. |
| BR-06 | Аллергии являются необязательными и сохраняются в профиле. |
| BR-07 | Оценка шефа доступна только после завершения класса. |
| BR-08 | Публичный рейтинг — среднее арифметическое оценок. |
| BR-09 | Фильтры работают по И-логике. |
| BR-10 | Авторизация осуществляется по телефону и SMS-коду. |
| BR-11 | При отмене студией повторная запись запрещена. |
| BR-12 | Клиент может выбрать несколько аллергенов из справочника. |

## 7. Границы системы

| In Scope (MVP) | Out of Scope |
|---|---|
| Адаптивное веб-приложение | Нативные мобильные приложения |
| Авторизация по SMS | Социальные сети, email, биометрия |
| Каталог слотов с фильтрами | Подписки и уведомления о новых слотах |
| Бронирование, прокат, аллергии | Онлайн-оплата и промокоды |
| Отмена более чем за 12 часов | Штрафы, waitlist, переносы |
| Оценка шефов | Публичные отзывы |
| История бронирований и аллергии | Лояльность и реферальные программы |
| Отображение отмен студией | Push, email и SMS-уведомления |
| Интеграция с существующим API | Изменения бэкенда и админки |
| Интерфейс только на русском языке | Мультиязычность и локализация |
| Без соблюдения требований к доступности | Соблюдение требований к доступности WCGA |
| Поддержка Google Chrome (Desktop, Android) и Firefox (Desktop) | Поддержка остальных браузеров |
