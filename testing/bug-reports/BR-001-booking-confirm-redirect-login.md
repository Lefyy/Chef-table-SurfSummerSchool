# BR-001 — После подтверждения брони пользователь перенаправлялся на `/login`

## Краткое описание

Авторизованный пользователь после нажатия кнопки подтверждения брони на странице `/slots/{slotId}/booking` вместо экрана успешного бронирования попадал на `/login`.

## Предусловия

- Пользователь авторизован.
- В выбранном слоте есть свободные места.
- Открыта форма бронирования `/slots/{slotId}/booking`.

## Шаги воспроизведения

1. Открыть доступный слот.
2. Перейти к бронированию.
3. Выбрать опции или оставить их пустыми.
4. Нажать «Подтвердить бронь».

## Фактический результат

После отправки формы пользователь перенаправлялся на `/login`, хотя сессия была активна.

## Ожидаемый результат

Бронь создаётся, пользователь перенаправляется на `/booking/success`.

## Приоритет и серьёзность

- Priority: Critical
- Severity: Critical

## Причина

POST-запрос подтверждения брони обрабатывался защитной конфигурацией как требующий CSRF-токен. Из-за отсутствия корректной CSRF-настройки для server-rendered формы запрос отклонялся, а пользовательский поток выглядел как потеря авторизации.

## Исправление

Баг решён изменением конфигурации безопасности: для POST-маршрута `/slots/*/booking` добавлено исключение из CSRF-проверки, при этом остальные защищённые страницы продолжают требовать авторизацию. После изменения подтверждение брони корректно создаёт бронь и ведёт на `/booking/success`.

## Статус

Resolved.
