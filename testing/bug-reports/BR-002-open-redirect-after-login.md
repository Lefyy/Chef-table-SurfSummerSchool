# BR-002 — Login принимал небезопасный внешний redirect

## Краткое описание

Параметр `redirect` на странице входа мог содержать внешний URL вида `//evil.test`, что создавало риск open redirect после успешной авторизации.

## Предусловия

- Пользователь не авторизован.
- Открыт URL `/login?redirect=//evil.test`.

## Шаги воспроизведения

1. Открыть `/login?redirect=//evil.test`.
2. Ввести валидный телефон.
3. Ввести корректный SMS-код.
4. Отправить форму.

## Фактический результат

После успешного входа приложение могло использовать небезопасный redirect-параметр.

## Ожидаемый результат

Приложение разрешает только внутренние относительные пути, начинающиеся с одного `/`, и игнорирует значения, начинающиеся с `//`.

## Приоритет и серьёзность

- Priority: High
- Severity: Major

## Причина

Недостаточно строгая проверка параметра `redirect`: проверялось начало со слэша без отдельной блокировки protocol-relative URL.

## Исправление

Добавлена нормализация redirect-параметра: значение принимается только если начинается с `/` и не начинается с `//`. При небезопасном значении пользователь после входа отправляется на безопасный маршрут по умолчанию `/schedule`.

## Статус

Resolved.
